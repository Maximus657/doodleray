## 1. v2rayN (Windows)

**GitHub:** https://github.com/2dust/v2rayN  
**Общая оценка:** Активно развивается, самый популярный Windows-клиент, но регулярно ломающийся после обновлений  
**Последний релиз:** ~v7.16.4 (февр.-март 2026)

### 🔴 Критические баги (блокирующие работу)

| # | Проблема | Частота | Источник | Период | Статус |
|---|---------|---------|----------|--------|--------|
| 1 | **TUN-режим вызывает «startup deadlock» в sing-box** — v2rayN генерирует remote rule-sets с `download_detour: "proxy"`, создавая петлю: sing-box пытается загрузить правила через прокси, который сам идёт через sing-box. Результат — полный отказ подключения | Часто | GitHub Issues (v1.19+) | 2025 | Открыт / Рецидивирует |
| 2 | **Полная потеря интернета на Windows 11 24H2 после перезагрузки** — HttpClient failures, «connection forcibly closed», «connection refused» для локальных портов, даже когда порты открыты | Часто | GitHub (июнь 2025) | 2025 | Частично исправлен |
| 3 | **v2rayN перестаёт работать после обновления на Windows 11 Pro 25H2** — тот же сервер работает на других устройствах и старых версиях v2rayN | Средне | GitHub (февр. 2026) | 2026 | Открыт |
| 4 | **v2rayN v7.12.5 массово перестал работать** — режим TUN выходит из строя через несколько часов, пользователи вынуждены откатываться | Часто | GitHub, ntc.party, Reddit | 2025 | Исправлен в 7.13+ |
| 5 | **Не работает при включённом Hyper-V / Core Isolation** на Windows 11 Pro — ошибка «failed to listen TCP on 9090» | Средне | GitHub (февр. 2024) | 2024 | Открыт |
| 6 | **DNS Deadlock в TUN-режиме** — клиент пытается отправить DNS-запросы через туннель, который ещё не установлен | Часто | GitHub (постоянно) | 2024-2026 | Обходное решение |

### 🟡 Серьёзные недоработки

| # | Проблема | Частота | Источник | Статус |
|---|---------|---------|----------|--------|
| 1 | **DNS-утечки при системном прокси через tray icon** — DNS-запросы уходят мимо прокси | Часто | GitHub | Открыт |
| 2 | **Split tunneling не работает в TUN-режиме на Win11** — правила routing с geosites/geoip игнорируются | Средне | GitHub | Открыт |
| 3 | **Импорт подписок Mihomo (Clash Meta) формата ломается** на v7.14.12+ из-за строгого парсинга | Средне | v2rayn-g.com (окт. 2025) | Открыт |
| 4 | **Окно «Shared server» зависает** на Windows 11 | Средне | GitHub (май 2024) | Открыт |
| 5 | **Баг с маршрутизацией для китайских пользователей** — удалённые routing rules автоматически восстанавливаются при закрытии окна настроек (v7.16.4) | Единично | GitHub | Открыт |
| 6 | **Замедление доступа к локальным сайтам** — версии 6.25/6.33+ заметно медленнее обращаются к внутренним сайтам (Taobao, Baidu) из-за встроенных DNS-правил | Средне | GitHub (февр. 2024) | Открыт |
| 7 | **Встроенный real-connection тест не работает** с новыми версиями ядер | Средне | v2rayn-g.com | Открыт |
| 8 | **VLESS без Flow объявлен deprecated** — предупреждение в логах, миграция на VLESS+Flow+Seed | Информационно | GitHub (февр. 2026) | Планируется удаление |
| 9 | **Ошибки обновления подписок (SSL errors)** — в v7.7.1+ нарушен SSL handshake, v7.5.6 работает нормально | Средне | GitHub (февр. 2025) | Частично исправлен |

### 🔵 UX/UI жалобы

| # | Жалоба | Частота | Источник |
|---|--------|---------|----------|

### 🔒 Безопасность

- **CVE:** Официально опубликованных CVE не обнаружено
- **VirusTotal флаги:** В 2024 году `v2rayN-With-Core.zip` с официального GitHub был помечен множественными антивирусами — подозрения на credential access, модификацию реестра, dropped executables включая **Xmrig криптомайнер**. Issue #4887 и #4832 — закрыты без объяснений
- **Security Policy:** Отсутствует (нет SECURITY.md)
- **Подпись бинарников:** Не подписаны (нет code signing)
- **Подозрения:** Серьёзные подозрения в supply chain compromise в 2024 — вопрос закрыт без ответа от разработчиков

## 📊 АНАЛИТИЧЕСКИЕ ВЫВОДЫ

### Какой клиент чаще всего ругают и за что?

1. **v2rayN** — чаще всего ругают за TUN-баги, DNS deadlocks, ломающиеся обновления, и сложный UI для новичков
2. **NekoRay** — ругали за memory leaks и нестабильность, но после архивирования ругать перестали — просто уходят
3. **Clash Verge Rev** — за уязвимость безопасности и «больше багов, чем у Clash for Windows»

### Какой клиент считают самым стабильным на Windows?

По данным сообществ:
1. **Happ** — для обычных пользователей (простой, работает из коробки)
2. **v2rayN** — для продвинутых (при правильной настройке стабилен)
3. **Hiddify** — для промежуточного уровня (красивый UI, но autostart проблемы)

### Какие фичи чаще всего просят?

1. **Kill switch** (при обрыве VPN — блокировать весь трафик)
2. **Per-app split tunneling** (routing на уровне приложений)
3. **Автоподключение при загрузке** (autostart + auto-connect)
4. **Встроенный скоростной тест** (точный, а не приблизительный)
5. **Человеческая документация на русском/английском** (не только на китайском)
6. **Автопереключение на рабочий сервер** при обрыве

### Есть ли клиент, который решает большинство жалоб?

**Нет.** Ни один клиент не решает все проблемы. Ближе всего к «идеалу»:
- **Happ** — для простоты (но закрытый код)
- **Clash Verge Rev** — для гибкости (но уязвимость безопасности в истории)
- **Hiddify Next** — для баланса (но нестабильный autostart)

### Какие проблемы специфичны для Windows?

1. TUN-драйверы (WinTUN) конфликтуют с Hyper-V и Core Isolation
2. Windows Defender false positives для ВСЕХ клиентов
3. UWP-приложения игнорируют системный прокси
4. Windows 11 24H2/25H2 обновления ломают сетевой стек
5. Необходимость VC++ Redistributables для Qt-клиентов
6. Права администратора (UAC) для TUN

### Тренд миграции пользователей (последние 6 месяцев)

```
Qv2ray (мёртв) ───────────────────┐
                                    ├──→ v2rayN, Hiddify, Clash Verge Rev
NekoRay (архивирован март 2025) ──┘
                                    
Clash for Windows (мёртв) ────────→ Clash Verge Rev, FLClash

v2rayN (часть пользователей) ─────→ Hiddify (из-за UI)
                                ──→ Happ (из-за простоты)

Clash Verge Rev (часть) ──────────→ FLClash (из-за багов)
```

**Набирающие популярность:**
- **Happ** — за простоту и кросс-платформенность
- **FLClash** — как замена багистого Clash Verge Rev (китайское сообщество)
- **Hiddify** — за UI и активную разработку

### Стало лучше или хуже за последние 6 месяцев?

**Смешанно:**
- ✅ **Лучше:** v2rayN активно исправляет баги, Hiddify улучшает UI, Clash Verge Rev закрыл уязвимость
- ❌ **Хуже:** NekoRay мёртв, Windows 11 24H2/25H2 ломает клиенты, VLESS deprecated, блокировки усиливаются
- ⚠️ **Не изменилось:** DNS-утечки, TUN-проблемы, отсутствие kill switch — остаются нерешёнными архитектурно
